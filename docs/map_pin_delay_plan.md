## 1. DB 변경

```sql
-- community_posts에 지연 시간 컬럼 추가
ALTER TABLE community_posts
  ADD COLUMN IF NOT EXISTS map_display_delay INT NOT NULL DEFAULT 0;
  -- 0 = 즉시, 1~120 = N분 후 지도 표시 (분 단위)
```

## 9. 꽃 식별 탭 (독립 기능)

게시글 올리지 않고 **혼자 꽃을 찾아볼 수 있는 전용 화면**.

### 위치
- 하단 탭에 추가 or 도감 탭 안에 버튼으로 진입

### 기능
1. 카메라 촬영 또는 갤러리에서 사진 선택
2. Plant.id API로 꽃 자동 인식
3. 인식 결과 표시:
   - 꽃 이름 + 신뢰도
   - 우리 DB(`flower_book`)에서 해당 꽃 정보 연결 → 꽃말, 키우는법, 개화시기 표시
   - DB에 없으면 Plant.id 결과만 표시
4. 결과 화면 하단 버튼 2개:
   - "게시글 쓰기" → `CreateFlowerSpotScreen`으로 이동 (사진 + 꽃이름 자동 입력)
   - "도감에서 보기" → `FlowerBookPage`로 이동 + 해당 꽃 이름으로 자동 검색

### Plant.id 활용
- 백엔드 `/api/v1/flower-spots/identify` 이미 구현됨
- 갤러리 사용 가능 (게시글과 달리 GPS 불필요)

### 구현 범위
- `FlowerIdentifyScreen` 신규 화면
- 하단 탭 또는 도감에서 진입
- `flower_book` DB 매칭 로직 (학명 기준)

## 10. 미등록 꽃 자동 도감 추가 (위키피디아)

Plant.id가 인식했는데 `flower_book`에 없는 꽃을 위키피디아에서 자동 수집해 DB에 추가.

### 흐름
```
Plant.id 인식 → flower_book에서 학명 검색
    ↓ 없으면
위키피디아 한국어 API로 꽃 정보 수집
    ↓
flower_book에 자동 저장 (source: 'WIKIPEDIA', status: 'AUTO')
    ↓
관리자가 나중에 수동 보완 가능 (status: 'COMPLETE')
```

### 수집 내용 (위키피디아)
- 꽃 이름 (한국어 일반명)
- 설명 (`extract` 필드)
- 썸네일 이미지 → Oracle Storage 저장
- 학명 (Plant.id 결과 그대로)

### 위키피디아 API
```
GET https://ko.wikipedia.org/api/rest_v1/page/summary/{꽃이름}
→ title, extract, thumbnail.source
```

### 주의
- 이미 flower_book에 있으면 건너뜀 (중복 방지)
- 위키피디아에도 없으면 "기타"로만 저장
- 꽃말, 키우는법은 없을 수 있음 (자동 수집 한계)

### 카테고리 매핑
- 학명 첫 단어(속명)로 `flower_species_mapping` 테이블 참조
- 매핑 없으면 "기타" 카테고리

## 11. 나만의 꽃 지도 (개인 꽃길 도감)

공유 없이 나만 볼 수 있는 개인 꽃 위치 기록 공간.

### 개념
- 공개 커뮤니티와 완전 분리된 **나만의 비공개 꽃 지도**
- 꽃을 발견했을 때 위치와 사진을 기록해두는 개인 아카이브
- 다시 찾아갈 수 있도록 지도에 내 핀만 표시

### 기능
1. 사진 + 위치 + 메모 기록 (공개 X)
2. 내 기록만 보이는 개인 지도 뷰
3. 날짜별/꽃 종류별 필터
4. "이 장소 다시 가기" → 길찾기 연결

### DB 설계
```sql
CREATE TABLE my_flower_spots (
    id          BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id     BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    plant_name  VARCHAR(100),          -- 꽃 이름 (Plant.id or 직접 입력)
    image_url   VARCHAR(1024),
    latitude    DOUBLE PRECISION,
    longitude   DOUBLE PRECISION,
    address     VARCHAR(255),
    memo        TEXT,                  -- 개인 메모
    visited_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### 차이점 (커뮤니티 vs 나만의 꽃 지도)
| | 커뮤니티 | 나만의 꽃 지도 |
|---|---|---|
| 공개 여부 | 공개 | 나만 |
| 지도 표시 | 전체 지도 | 내 지도만 |
| 알림 | 근처 사용자 알림 | 없음 |
| 댓글/좋아요 | 있음 | 없음 |

### 진입점
- 내정보 화면 또는 하단 탭
