# PR-D 커뮤니티 읽기 Evidence 정리

## 문제
커뮤니티 게시글에는 제목이 없고 글, 사진, 꽃 이름, 위치 정보 중심입니다. 그런데 챗봇이 게시글을 제목이 있는 콘텐츠처럼 다루면 검색 품질과 답변 품질이 어색해집니다. 또한 결과 0건이 오류처럼 보이면 사용자 경험이 나빠집니다.

## 목표
커뮤니티 읽기 flow를 제목 없는 게시글 구조에 맞춥니다. 결과 0건, 조회 실패, 검색어 없음, 최신/인기 요청을 명확히 구분합니다.

## 검색 기준
```text
content
flowerSpecies
plantName
address
createdAt
likes
comments
```

## 구현 계획
- `community.searchPosts`는 제목이 아니라 본문/꽃명/식물명/주소를 검색합니다.
- `community.getLatestPosts`, `community.getPopularPosts`는 검색어가 없으면 전체 기준으로 조회합니다.
- 결과 0건이면 `SUCCESS + items=[]`로 유지합니다.
- DB 조회 실패만 `ERROR`로 처리합니다.
- Answer AI에는 글 내용 요약만 전달하고 `title` 필드를 만들지 않습니다.

## 답변 예시
```text
현재 수국 관련 후기는 아직 없습니다. 커뮤니티에 새 후기를 남기거나 다른 꽃 이름으로 찾아볼 수 있어요.
```

```text
최근 커뮤니티 글은 아직 없습니다.
```

## 테스트
- `수국 후기 찾아줘` → 검색 결과 0건이면 없음 답변
- `최신 글들은 어떤 걸 소개 해?` → 최신글 도구, keyword 빈 값
- `이번 주 인기글 보여줘` → 인기글 도구, 기간 필터
- `장미 게시글 검색해줘` → content/꽃명 기준 검색

## 완료 기준
- `응답을 가져오지 못했습니다.`가 결과 없음 상황에서 나오지 않습니다.
- 제목 기반 표현이 답변/도구 컨텍스트에 나오지 않습니다.
- 최신/인기/검색 도구가 서로 섞이지 않습니다.
