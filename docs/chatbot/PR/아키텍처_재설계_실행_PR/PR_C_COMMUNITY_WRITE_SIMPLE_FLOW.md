# PR-C 커뮤니티 게시물 작성 단순 흐름

## 문제
이전 설계는 `수국 후기 글 써줘`를 꽃 자체 후기인지, 장소 후기인지, 축제 후기인지 복잡하게 확인하려 했습니다. 하지만 v1에서는 사용자가 게시물을 작성하려는 의도가 명확하면 작성 화면만 열면 됩니다.

## 목표
커뮤니티 글 작성 의도는 복잡한 대상 검증 없이 `COMMUNITY_COMPOSE` action으로 연결합니다. 자동 작성/자동 게시 없이, 자연스러운 안내 답변만 제공합니다.

## 적용 문장
```text
수국 후기 글 써줘
커뮤니티에 글 올릴래
후기 글 작성하고 싶어
꽃 본 거 올리고 싶어
서울식물원 후기 글 써줘
```

## 구현 계획
- Route AI에 `community_write` flow를 명확히 둡니다.
- `글 쓰기`, `글 올리기`, `후기 작성`, `올리고 싶어`는 `community_write`로 보냅니다.
- 서버는 `COMMUNITY_COMPOSE` action만 생성합니다.
- Flutter action은 실제 최신 작성 화면인 `CreateFlowerSpotScreen`으로 연결합니다.
- 답변은 짧고 자연스럽게 만듭니다.

## 답변 원칙
좋은 예:
```text
후기 작성 화면을 열었어요. 사진과 함께 본 장소나 느낌을 편하게 남겨보세요.
```

나쁜 예:
```text
수국에 대한 후기 글 작성 화면을 열어드리겠습니다. 화면을 열었습니다.
```

## 금지
- 게시글 본문 자동 생성 금지
- 사진 자동 선택 금지
- 자동 게시 금지
- 장소/축제 대상이 부족하다고 되묻기 금지

## 테스트
- `수국 후기 글 써줘` → `NAVIGATE COMMUNITY_COMPOSE`
- `커뮤니티에 글 올릴래` → `NAVIGATE COMMUNITY_COMPOSE`
- `글 내용까지 대신 저장해줘` → `app.unsupported`, action 없음
- Flutter에서 실제 작성 화면이 열리는지 확인

## 완료 기준
- 게시물 작성 의도는 작성 화면으로 이동합니다.
- 답변이 중복되거나 딱딱하지 않습니다.
- 자동 작성/게시가 발생하지 않습니다.
