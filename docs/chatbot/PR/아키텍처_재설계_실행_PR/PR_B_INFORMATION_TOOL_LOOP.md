# PR-B 정보성 도구 반복 구조

## 문제
정보성 질문은 한 번의 도구 호출로 충분하지 않을 수 있습니다. 예를 들어 `수국 꽃말이랑 언제 피는지 알려줘`, `장미 특징이랑 키우는 법 알려줘`처럼 복합 질문은 여러 정보 조각이 필요합니다. 반대로 지도/작성/단순 이동은 반복 호출이 필요 없습니다.

## 목표
정보성 flow에만 도구 반복을 허용합니다. 반복은 **최소 1회, 최대 2회**로 제한합니다.

## 적용 flow
```text
flower_information
community_read
festival_information
```

## 구현 계획
- Role Planner가 첫 번째 도구와 필요한 정보 슬롯을 정합니다.
- Tool Executor가 1회 실행합니다.
- Evidence Check가 충분/부족/없음/오류를 판단합니다.
- 부족하지만 보완 가능한 경우에만 2번째 도구를 호출합니다.
- 2회 후에는 부족해도 답변을 생성하고, 없는 정보는 없다고 말합니다.

## 도구 반복 예시
```text
수국 꽃말이랑 언제 피는지 알려줘
→ flower.getMeaningAndBloom 1회
→ 충분하면 답변
```

```text
장미 특징이랑 키우는 법 알려줘
→ flower.getBasicInfo 1회
→ growTips 부족 판단
→ flower.getGrowGuide 1회 추가
→ 답변
```

## 금지
- 지도 질문에서 반복 도구 호출 금지
- 커뮤니티 작성에서 반복 도구 호출 금지
- 결과 없음인데 억지로 다른 도구를 계속 호출 금지
- 3회 이상 도구 호출 금지

## 테스트
- `수국 꽃말이랑 언제 피는지 알려줘` → 1회 또는 2회, 2회 초과 없음
- `장미 특징이랑 키우는 법 알려줘` → 기본 정보 + 재배 정보 조합 가능
- `수국 후기 찾아줘` → 커뮤니티 검색 1회, 결과 0건이면 없음 답변
- `벚꽃 지도에서 보여줘` → 반복 호출 없음

## 완료 기준
- 정보성 질문만 tool loop를 사용합니다.
- 도구 호출은 최대 2회입니다.
- 없는 정보는 생성하지 않습니다.
